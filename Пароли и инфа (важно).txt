Файл в корне .htaccess (главный) переименовываем в .htaccess и заливаем на хост с трекером и меняем:
В каждой папке есть свой .htaccess желательно ему выдавать права 555 самые низкие, то есть только для чтения!!!
ErrorDocument 404 localhost/404.php
ErrorDocument 403 localhost/403.php
ErrorDocument 500 localhost/500.php
где localhost - ваша ссылка на трекер - тоже обязательно, т к будет показ о ошибах на вашего трекере, а не страничка браузера.

Файл include/passwords.php идет самый главный для настроек, там:
настройка количество пользователей, приглашений, тема по умолчанию (для гостей и пользователей первых), имя почты (она должна быть прописанна также в functions.php вначале), 
имя сайта и тд.


Настройка почты для отправки подтверждения пользователю о активации аккаунта:

Зайти в файл config.php найти и изменить на вашу почту, строчку - 
$SITEEMAIL = "ваша @ почта";

также эта же почта (ОБЯЗАТЕЛЬНО) прописывается в passwords.php строчка:

            $accountname="ваша @ почта";  // пример login@bossmail.ru
            $accountpassword="password"; // ваш пароль на эту почту
            $smtp_host="smtp.bossmail.ru";  // smtp почт сервер для вашей почты, smtp.bossmail.ru пример.
/// если на домене есть своя почта, то вписываем localhost выше вместо smtp.bossmail.ru и тд
            $smtp_port="25";  /// порт на почт сервер, см у провайдера или же хоста


PS: TLS - нет настроек пока, используйте почту без защищенного протокола tls.


В папке pic/watermark_dark.png и watermark_light.png файла редактируем на фотошопе, сохраняем в том же файле, они используются для наложения картинки при заливки торрента файла изображения.


После настройки почты (если точно знаете что настроили и работает) меняем в файле config.php строчку:

$use_email_act = 0; // Использовать активацию по почте, иначе - автоматическая активация при регистрации.
на
$use_email_act = 1; // Использовать активацию по почте, иначе - автоматическая активация при регистрации.



При чистом трекере ошибка может быть только одна с тегами, это
Warning: Invalid argument supplied for foreach() in D:\xampp\htdocs\include\functions.php on line 1653
она означает что нет тегов в раздаче, т е при заливке первой раздачи и с тегами внутри - пропадет ошибка.

Да и самое важное от взломов, в каждой папке есть файл .htaccess, он защищает вас от любых видов взлома, к примеру php sql кода в изображении, в папке только определенные виды расширения и разного другого вида лома.
